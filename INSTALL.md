# INSTALL — инструкция для агента Extella (как развернуть этот плагин)

Ты — агент Extella. Тебя попросили установить плагин «Визард внедрения» из этого репозитория
в аккаунт пользователя. Выполни шаги по порядку. Все операции — реальные вызовы платформы.
Ничего не выдумывай: если шаг не удался — честно сообщи и остановись.

## 0. Предпосылки
- Питон 3.12 на устройстве пользователя (для локального моста).
- Устройство зарегистрировано как target платформы (для пиннинга серверных экспертов).
- У пользователя есть PIN (6+ символов) для сейфа секретов (vault).

## 1. Эксперты (папка `experts/`)
Для каждого файла `experts/*.py`:
- имя эксперта = из шапки `# expert:` (или имя файла без `.py`);
- сохрани через `save_expert` c `cspl="fython"`, `global=true`,
  `description` из шапки `# description:`, `kwargs` — из шапки `# params:` (значения по умолчанию пустые).
Проверка: `list_experts` должен показать все 26 `wz_*`.

## 2. Концепты (папка `concepts/`)
Для каждого `concepts/<id>_<slug>.md`:
- тело после строки-заголовка → `concept_add` (с API-ключом эмбеддинга, иначе вектор не создастся);
- сохрани соответствие исходный id ↔ новый id (в этом аккаунте id будут свои).

## 3. Правила (папка `rules/`)
Для каждого блока `## rule_id:` в `rules/*.md` → `rules_add`. Это канон, которому обязаны следовать агенты.

## 4. Агенты (папка `agents/`) — ПЛАТФОРМЕННЫЙ QWEN
> Решение владельца от 20.07.2026: клиентских агентов **можно создавать через API**, если
> провайдер и модель заданы явно как платформенные `alibaba` / `qwen3.7-max-2026-06-08`.
> Запрещены `agent_extella_default` (платный Claude), `provider=custom` и BYOK-модели. Нельзя
> использовать generic `agent_create` с неявными дефолтами: сначала убедись, какой payload он отправляет.

Порядок:
1. Создай двух агентов прямым `POST /api/agent/create` с явными `provider=alibaba` и
   `model=qwen3.7-max-2026-06-08`: один под «Визард внедрения», второй под «Строитель».
   Если платформа не разрешила API-создание на этом аккаунте, фолбэк — пользователь дважды
   делает UI-копию базового Qwen и передаёт два `agent_id`.
2. Для каждого проверь через `/api/agent/get` на ЭТОМ аккаунте: id существует, provider=`alibaba`,
   model начинается с `qwen3.7`. Затем пропиши инструкции через **`agent_update`** из
   `agents/wizard_agent.instructions.md` и `agents/builder_agent.instructions.md`;
   привяжи к профилю пользователя, подключи эксперты `wz_*` и концепты/правила из шагов 1–3.
3. **Проверь keyless-запуск:** вызови `agent/run` с коротким `input` для агента-Визарда.
   Если вернулось `pro_key_required` или `Agent does not belong to this user` — не продолжай с
   этим id: повтори создание с явным Qwen-payload либо используй UI-копию как фолбэк.
4. Запиши id. **id Визарда идёт в `config.agent_id`** (шаг 5) — это одновременно keyless-бэкенд
   визарда (blueprint/план/кодоген идут на него). Никогда не ставь сюда `agent_extella_default` — это Claude (платно).

## 5. Интерфейс + локальный мост (папка `ui/`)
- разверни `ui/server.py` и `ui/wizard.html` в `~/extella_wizard/app/` на устройстве пользователя;
- создай `~/extella_wizard/app/config.json` с токеном пользователя, портом 8765 и
  `agent_id` = id Qwen-агента Визарда из шага 4 (это же keyless-бэкенд: на него идут blueprint/план/кодоген).
  Опционально `llm_agent_id` — отдельный Qwen под кодоген, если не хочешь грузить историю Визарда;
  оставь `llm_api_key` пустым (keyless на Qwen). НИКОГДА не ставь `agent_extella_default` — это Claude;
- поставь автозапуск (LaunchAgent на macOS / systemd --user на Linux) — можно экспертом `wz_install_autostart`;
- зарегистрируй плагин в тулбаре по образцу `extella-plugin.json` (секция `ui`/`service`).
Проверка: `GET http://127.0.0.1:8765/x/health` отдаёт `{"status":"ok"}`.

## 6. Сейф секретов (vault)
Спроси у пользователя PIN и выполни провижининг ключа (`/x/vault_provision` или эксперт
`wz_vault_provision`) — так секреты (Telegram/CRM/БД) будут шифроваться ключом, который есть
только на его устройстве.

## 7. Финальная проверка
- открой `http://127.0.0.1:8765` → вкладка «Автоматизации» → блок «🔌 Подключения» виден;
- `GET /x/monitor` не даёт ошибок;
- сообщи пользователю: «Визард развёрнут, откройте Автоматизации».

## Обновление (повторный запуск)
Повтори шаги 1–3 (перезапишут эксперты/концепты/правила) и 5 (обновит мост). Данные пользователя
(сессии, секреты, расписания) не трогай.
