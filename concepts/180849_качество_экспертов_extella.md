# concept_id: 180849
# tag: wz-методология

[wz-методология] Качество экспертов Extella (из инструкции платформы §4). Эксперт = переиспользуемый актив, а не одноразовый скрипт. Пять обязательных элементов: директива $extens("include.py") первой строкой; зависимости через include(..., ["extella-pip install ..."]) (никогда голый pip); сигнатура с явными типами и дефолтами, БЕЗ *args/**kwargs; валидация входов с ранним возвратом ошибки; возврат всегда dict со status. Четыре абсолютных запрета: хардкод путей/ключей/ID (всё параметрами); *args/**kwargs; возврат бинарных данных (только пути к файлам); прямой доступ эксперта к KV Store изнутри кода — credentials получает АГЕНТ через kv_get и передаёт параметрами при запуске (эксперт = чистая логика без скрытых зависимостей от облака). Description — это поисковый индекс, не комментарий: одно предложение что делает + ВСЕ параметры с назначением; плохое описание = эксперт не найдётся семантическим поиском. Имена snake_case; сохранение с существующим именем перезаписывает без предупреждения — версии суффиксами (_v2). Клиентская специфика (колонки выгрузок, брендовые регексы, чек-листы, промпты) — всегда параметры/конфиги, не тело кода.
