# concept_id: 180850
# tag: wz-методология

[wz-методология] Декомпозиция процесса на проверяемые стадии (адаптация planning-and-task-breakdown, addyosmani/agent-skills, MIT). Хорошая декомпозиция — разница между процессом, который надёжно доводится, и клубком. Правила: (1) Сначала карта зависимостей: что от чего зависит (данные → обработка → оценка → отчёт); независимые ветки помечай — их можно строить/запускать параллельно. (2) Каждая стадия — маленькая, с явным критерием приёмки («на выходе файл X с полями Y», «оценено N единиц, у каждой есть цитата»); стадия без проверяемого выхода — плохо нарезана. (3) Тонкие вертикальные срезы вместо слоёв: сначала прогнать ОДну единицу через ВЕСЬ процесс до конца (end-to-end на образце), потом наращивать объём — а не строить все стадии «вширь» без единого сквозного прогона. (4) Упавшая стадия останавливает только зависимые от неё, независимые продолжаются. (5) Первый сквозной прогон на синтетике/образце — раньше любых объёмов. В Extella это ложится на оркестратор с манифестом стадий.
