# concept_id: 180852
# tag: wz-методология

[wz-методология] Уточнение размытой бизнес-идеи (адаптация idea-refine, addyosmani/agent-skills, MIT). Когда клиент приходит с сырой идеей («надо что-то с ИИ для отдела»), не суживай преждевременно. Два такта: ДИВЕРГЕНЦИЯ — переформулируй идею своими словами, задай заостряющие вопросы, предложи 3–5 вариантов решения одной боли (в т.ч. более простой и более амбициозный, чем просил клиент); КОНВЕРГЕНЦИЯ — сгруппируй варианты, стресс-тестируй допущения («что должно быть правдой, чтобы это сработало?»), выбери направление. Итог — one-pager: Проблема (одним абзацем, с ценой проблемы в деньгах/часах) · Рекомендованное направление · Ключевые допущения (проверяемые) · Объём MVP (минимум, который уже полезен) · Чего НЕ делаем (граница объёма — самый важный раздел). Философия: простейшая версия, которая всё ещё решает настоящую проблему. В Extella one-pager = вход для Process Blueprint.
