# tag: wz-методология

[wz-методология] Blueprint раньше стройки (адаптация spec-driven-development, addyosmani/agent-skills, MIT). Строить процесс без спецификации = гадать. Гейтовый порядок: СПЕЦИФИКАЦИЯ → ПЛАН → ЗАДАЧИ → РЕАЛИЗАЦИЯ, на каждом гейте — ревью человеком (владельцем процесса), дальше без валидации не идём. Правило допущений: ДО написания blueprint перечисли явно, что ты предполагаешь («данные выгружаются в Excel», «чек-лист уже утверждён», «ПДн выносить нельзя») и попроси поправить — молчаливые допущения = самая опасная форма недопонимания. Blueprint процесса в Extella обязан содержать: имя и цель (что получит бизнес), пригодность и риск (по рубрике самообслуживания), стадии (каждая — из возможностей каталога платформы, без выдуманных компонентов), честные разрывы (чего платформа пока не умеет + как закрываем), план теста на образце с критериями успеха, открытые вопросы. Когда НЕ нужен полный цикл: разовая мелкая задача одним модулем — делается сразу.
