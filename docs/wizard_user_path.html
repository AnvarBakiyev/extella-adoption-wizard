Рамка и канон

Extella Adoption Wizard: «человек превращает бизнес-боль в работающий ИИ-процесс» — строительная спецификация пользовательского пути (этапы 0–7). Кому: Анвар (со-основатель/CCO Extella.ai) и команды-исполнители (домен визарда передаётся разработчику Тимуру). Документ — снимок «как построено» + точечный долг, пара к DEV_HANDOFF.md (техническая вводная) и docs/WIZARD_ARCH_TZ.md (ТЗ кабины).

База (as-built). Продукт — три слоя: UI ui/wizard.html (единый HTML, без бандла) ↔ мост ui/server.py (локальный HTTP на 127.0.0.1:8765, версия BRIDGE_VERSION 3.75, эндпоинты /x/*, держит токен Extella) ↔ платформа: 42 эксперта wz_*/kp_* + агенты Qwen + KV. Живые агенты: Визард agent_hM0qLHwu-Hw_4sjydTU1g (интервьюер), Строитель agent_FLYxB0v1qIY2phB5beVP5 (стройка). Планировщик — VPS PS.kz cron */15 * * * * tick.py (wz_scheduler_tick). Доказано на живых кейсах: «Дебиторка 28 филиалов» (ET-Tech), пак Travel Agency (8 экспертов), флагман Competitor Intelligence.

Легенда статусов: есть — работает сегодня, доказано; частично — работает не целиком или требует сверки/живых кредов; построить — отсутствует. Рабочие имена новых операций помечены предложение.

Инварианты (канон платформы, нарушать нельзя ни одной стройкой):

  • Клиентская LLM — только платформенный Qwen, никогда не Claude. Диспетч по agent_id в теле через qwen_agent(), не через заголовок. BYOK/gpt-4o запрещены.
  • Продовые агенты заморожены (F2). Изменения процесса — только через Строителя с записью решения в сессию (~/extella_wizard/sessions/).
  • Письма и внешняя запись — только черновики, отправляет человек (коннекторы шлют по явной настройке получателей).
  • Агенты по API: правило «один инструмент за ход» (баг склейки аргументов); длинные задачи дробить (лимит ~50 итераций/ход); HTTP 500/timeout ≠ провал — проверять по артефактам; сообщения самодостаточны (previous_response_id теряется).
  • Общий KV — только с global:true (шаренные каталоги composer:catalog, _mkt_automations); per-account ключи (flow:, sec:, сессии) — без флага.
  • Скоупинг: эксперты/концепты изолированы per-agent; один expert_name в нескольких скоупах = недетерминированный запуск.
  • Секреты (токен Extella, креды коннекторов) — из ~/extella_wizard/app/config.json / vault, в логи и чаты не печатать (_scrub).

Отличие от Workspace-подпроекта. Workspace документировал в основном «что построить»; здесь — обратная картина: этапы 0–7 в массе есть и доказаны. Долг точечный (раздел 14): пересборка построенного процесса по правкам полей, B2-вебхук (отложен Анваром), хвост EN-локализации, живые креды части источников.

Ключевая мысль. Композитор и Мастер — две скорости одной платформы: Композитор собирает из готового за минуты (задача), Мастер строит новое за дни (бизнес-процесс). Обе двери сходятся в один кабинет (этап 7), где процесс живёт: запуск, расписание, доставка, правки. Это и есть compounding: каждый построенный процесс либо переиспользует блоки, либо добавляет один — следующий собирается быстрее.

3.2. Жизненный цикл сессии (единая нить)

Сессия — стержень пути; её поле stage двигается: interview → blueprint → built → (launched при расписании). Одна сессия связывает цепочку интервью → blueprint → план → builds[] → schedule → recipients → runs, что даёт кабинету (этап 7) честный ответ «из чего собран этот процесс и что он делал». /x/automations строит витрину именно из сессий с builds (не из KV-экспертов).

Канон записи. Сессии — источник решений: любое изменение процесса Строитель пишет в сессию. Продовый агент заморожен (F2) — правка идёт через пересборку, не «руками на живом».

15. Открытые вопросы Анвару

  • Приоритет долга. Что важнее для ближайшей цели — пересборка по полям (W1, делает кабинет «настоящим») или US-хвосты (W4)? По умолчанию рекомендую W1.
  • B2-вебхук (W3). Разморозить сейчас или держим на polling до конкретного клиента с событийным входом?
  • Пак-шаблоны (W5). Нужен ли вход «поставил отраслевой пак → лёгкая адаптация» как отдельная дверь, или достаточно Композитора+Мастера?
  • Архетипы (W6). Фиксируем словарь архетипов конфигом (правится без перепрошивки) или оставляем Qwen-детект как есть?
  • Граница «средней» правки. Где проходит грань «правлю в кабинете» vs «ухожу в Мастера»? Сейчас: поля/правила/шаблон/расписание — в кабинете; логика — в Мастера. Подтвердить.
  • Внешняя раздача паков. Публикация в витрину/установка коллегам — заморожена до твоего прямого слова (подтверждено с тулбар-стороной). Оставляем заморозку?
