# Фаза 1 — разбор монолита `server.py` (безопасная процедура)

Статус: план от 07.07.2026. Это **не «переложить код»**, а **модуляризация самообновляющегося
демона на парке мостов**. Делать по этой процедуре, иначе — кирпич на фронте.

## ⚠️ Главный риск (прочитать первым)
Мост обновляется сам: `publish_release.py` кладёт в подписанный KV-канал `rel:bridge:*` **ровно два
файла — `server.py` и `wizard.html`** (хардкод, `for name, raw in [("server.py",…),("wizard.html",…)]`).
Мосты с `auto_update:true` тянут манифест, проверяют подпись (Ed25519, публичный ключ вшит в server.py),
применяют файлы и рестартуют.

**Следствие:** если разнести `server.py` на модули и НЕ тронуть релиз — новый `server.py` в канале
импортит отсутствующий модуль → ImportError при рестарте → авто-откат (в лучшем случае) или кирпич.
Поэтому разрез идёт **в связке с релиз-механизмом**.

## Целевая раскладка модулей (`ui/bridge/`)
| Модуль | Забирает из server.py |
|---|---|
| `platform.py` | `api()`, `run_expert()`, `parse_expert_result()`, `HEADERS`, `BASE`, поллинг `tasks/check` |
| `llm.py` | `qwen_agent()`, `_gen_identity()`, вызовы `agent/run` |
| `build.py` | `_run_build`, `_build_one`, `_audit_experts`, `_ORCH_TEMPLATE`/`_make_orchestrator`, kp-стадии |

| `ops.py` | релиз/авто-апдейт (подпись, apply, rollback), `/x/health` |
| `server.py` | тонкий роутер `/x/*` + запуск |

## Безопасная последовательность (строго по шагам)
**Шаг 0 — подготовить релиз-механизм к многофайловости (как ОБЫЧНОЕ одно-файловое обновление):**
- `publish_release.py`: слать **всю папку** `bridge/*.py` (+ wizard.html), манифест перечисляет все файлы.
- Логика apply в мосту: применять **произвольный набор** `manifest.files` (не 2 хардкод-файла), писать
  каждый в `~/extella_wizard/app/`, компилировать, smoke, только потом рестарт; при провале — откат.
- Выкатить это как версию N (ещё single-file по сути) и **проверить на отдельном тестовом мосту**
  (другой порт + `~/extella_wizard_test/app`), что цикл обновления проходит чисто. ТОЛЬКО потом дальше.

**Шаг 1..k — резать по одному шву:**
- Вынести один модуль (начать с `platform.py`) → `server.py` импортит его.
- `python3 -m py_compile` всех файлов → `scripts/smoke.py` зелёный (против локального тест-моста нового
  формата) → задеплоить на тестовый мост → проверить, что обновление применилось и E2E жив → коммит.
- Повторить для `llm → build → connectors → ops`.

**Рычаги безопасности:**
- `python3 publish_release.py --halt` — стоп-кран канала (мосты перестают применять релиз).
- Авто-откат моста при неуспешном старте (`MAX_UPDATE_ATTEMPTS`).
- `scripts/smoke.py` после каждого шва.
