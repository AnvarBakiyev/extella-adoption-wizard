# Отчёт, который клиент правит СЛОВАМИ (замысел Анвара, 18.07.2026)

## Решение Анвара
Доводить шаблон до совершенства НЕ надо — «просто чтоб красиво». Ценность в другом:
клиент должен уметь **словами** сказать, каким он хочет отчёт («другая шапка», «другие
цифры», «таблицу свёрстывай иначе», «добавь график»), и получить это без нас.
Место — его личный кабинет (кабина автоматизации).

## Почему это НЕ стройка с нуля: три из четырёх частей уже есть
| Часть | Что это | Состояние |
|---|---|---|
| ЧТО в отчёте | `cspl_report_dsl` — декларативная спека: columns / filter / group_by | **есть** (CSPL Studio S1) |
| Правка словами | чат кабины + `/x/classify_intent` (понимает «шли на email», «еженедельно») | **есть** (C4) |
| КАК выглядит | оформитель PDF, чистый Python (`report_render_prototype.py`) | **прототип** (18.07) |
| Связка | фраза → правка спеки → следующий прогон рендерит по-новому | **НЕ СДЕЛАНО** |

## Что делать (порядок)
1. Расширить спеку отчёта видом: `title/subtitle/sections/chart` — сейчас в ней только данные.
2. Научить чат кабины распознавать намерение «поправить отчёт» и звать генеративный
   builder CSPL (он уже проектирует СПЕКУ, а не код: `/x/cspl_create`, locked_fields, fixtures-гейт).
3. Показывать превью ДО применения (гейт человека — как в адаптерах AC-05).
4. Хранить спеку в версиях с откатом (как адаптеры и правки кабинета).
5. Оформитель завернуть в эксперт `fmt_report_pdf` и подключить последним шагом оркестратора.

## Технические выводы прототипа (проверено вживую)
- **Chrome не годится**: на хостинге его нет, у клиента может не быть. Способность должна
  жить В ЭКСПЕРТЕ — чистый Python + `include("import fpdf", ["extella-pip install fpdf2"])`.
- **DM Sans (вариативный) НЕ РАБОТАЕТ**: нет кириллицы в базовом начертании, fpdf2 падает
  на переменных шрифтах. Нужен статический шрифт с кириллицей: Arial (macOS) → DejaVu (Linux)
  → Liberation; если ни одного — честный отказ, а не квадратики.
- **Никаких знаков Extella в документе**: он принадлежит клиенту и уходит ЕГО заказчикам.
  Шапка — имя владельца процесса. Наш бренд живёт в интерфейсе, а не в чужих отчётах.
- Шкала полос должна быть ОБЩЕЙ для всех разделов, иначе одинаковая длина означает разное.
