# Шум в панели «Что делает Extella» — диагноз и фикс

**Репорт Анвара (18.07):** в панели бесконечно повторяется «Обновляется список плагинов», чип
внизу показывает «✓ 999».

## Диагноз (не визард)

Источник — **Extella Activity Center**, а НЕ код визарда:
`~/Library/Application Support/Extella Activity Center/activity_model.py`
(ставится вместе с marketplace-паком, мост :8799, **не под git**).

Строки задаёт словарь `SEMANTICS` (`_etb_registry_read` → «Обновляется список плагинов»,
`wz_scheduler_tick` → «Проверено расписание»). В `build_activity` лента `history` собиралась
**по строке на КАЖДЫЙ запуск**, схлопывания повторов не было:

- «Проверено расписание» — **814** одинаковых строк
- «Задача: app list» — **59**
- «Обновлён список плагинов» — **36**

Флаг `recurring` в SEMANTICS влияет только на подпись режима, не на группировку.
Файлы реестра плагинов при этом НЕ переписываются (проверено по mtime) — это чисто логирование.

## Фикс (применён локально, проверен)

`build_activity`: перед срезом `history[:40]` схлопываем **буквально одинаковые** записи
(та же `function` + `status`) — держим самую свежую, число повторов дописываем в `detail`
и в поле `repeatCount`. Разные задачи остаются отдельными строками. Фронт менять не нужно.

Патч: `docs/handoff/activity_center_collapse_repeats.patch`

**Результат (проверено на живой ленте):** 39 строк вместо сотен, дублей нет:
`Проверено расписание (повторов: 814)`, `Задача: app list (повторов: 59)`,
`Обновлён список плагинов (повторов: 36)`.

## ЧТО НУЖНО СДЕЛАТЬ ДАЛЬШЕ

Правка живёт в `Application Support` и **будет затёрта при следующей установке/обновлении пака**.
Чтобы фикс жил — его надо внести в ИСХОДНИК Activity Center (контур установщика/Эллы), откуда
файл раскладывается. Бэкап оригинала рядом: `activity_model.py.bak_<timestamp>`.
