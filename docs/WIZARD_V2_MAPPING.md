## 6. Что требует владельца платформенного бэкенда (вопросы к команде api.extella.ai)

Эти вещи нельзя построить на стороне моста/визарда — нужны серверные сущности:

1. **Capability Registry / Model Registry / Execution Gateway** (§8.9) — единый каталог с манифестами
   и версиями. Сегодняшний субстрат: global-KV, но он скоупится по агенту (источник «теней»).
2. **Immutable Release + lockfile** (§8.9.10) — на платформе нет версионирования экспертов;
   `expert/save` перезаписывает. Semantic versioning экспертов = платформенная фича.
3. **Target Passport / heartbeat / attestation** (§13) — listener сегодня не публикует паспорт.
4. **Секреты как leases** (§20.6).
5. Известные платформенные баги, которые всплывут в v2: `agents/list` пуст, `agent/get` не отдаёт
   instructions, склейка аргументов («один инструмент за ход»), KV-скоуп по X-Agent-Id без явного
   «shared» неймспейса.
