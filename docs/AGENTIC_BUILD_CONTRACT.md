# Агентная стройка сложных процессов (as-built, v4.97)

## Когда включается

`wz_build._run_build` переводит задачу в `build_mode=agentic`, если у неё несколько одновременно
используемых файлов либо нелинейный DAG. Линейная одновходовая стройка сохраняет прежний путь.

## Контракт

1. `wz_agentic.make_task_package` сохраняет полный аудиторский пакет: ответы интервью, комментарии,
   решения, правила/поля, blueprint, build plan, все профили файлов и capability-каталог.
2. Для кодогенерации из него строится отдельный компактный `BUILD_BRIEF`: прямые указания владельца
   → согласованные blueprint/ТЗ → профили входов. Старый compiler-plan и стенограмма чата в промпт
   не попадают; каталог фильтруется до максимум шести релевантных capabilities.
3. Qwen создаёт один исполняемый CSPL/fython-эксперт с фиксированной сигнатурой.
4. Код не допускается к запуску, если содержит значения контрольного образца, абсолютные пути,
   shell или fallback-поиск вне `source_file`.
5. Визард, а не строитель Qwen, запускает эксперта с явными `source_file` и `output_dir`.
6. Детерминированная приёмка требует доказательства обработки каждого входного файла,
   содержательную summary, acceptance checks и открываемые `report.md`/`report.xlsx`.
7. Независимый Qwen-судья проверяет результат против того же `BUILD_BRIEF`. При ошибке Строитель
   получает категорию сбоя и агрегаты, но не готовую таблицу контрольных ответов; значения образцов
   редактируются. Максимум четыре попытки solve → run → repair.
8. Только принятая версия записывается в build с `agentic_contract=1` и может запускаться из кабинета.

Доказательства лежат в `runs/<build_id>/task_package.json` и `agentic_evidence.json`.

## Исполнение и упаковка

Каждый ручной прогон получает отдельную папку `runs/process_runs/<session>/<run_id>`; затем отчёты
переносятся штатным persistence-слоем. `/x/deploy_agent` синхронизирует в выбранного агента один пакет:
эксперты сборки, tagged concepts процесса и tagged rules владельца. Повторный deploy применяет diff.

Создание новой «копии Qwen» через `/api/agent/create` обязательно проходит реальный smoke-test.
Версия, провайдер, пользовательский ключ и OpenAI-compatible endpoint не фиксируются: сначала
копируется уже настроенный Qwen пользователя, Alibaba Qwen остаётся fallback. Если новая Pro-копия
требует BYOK (`pro_key_required`), она не считается рабочей и не попадает в реестр, но её id
сохраняется, чтобы пользователь мог привязать ключ/endpoint и добавить тот же агент.

## Честное ограничение v4.97

Локальный multi-file bundle работает вручную на устройстве владельца. Расписание 24/7 для такого
каталога блокируется кодом `local_bundle_schedule_unsupported`, пока не реализован device-local
планировщик либо материализация всего bundle на VPS.
