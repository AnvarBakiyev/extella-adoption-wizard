## Блокер: симптом

`scripts/live_wizard_agentic_e2e.py` (Excel+PDF сверка пломб) падает на шаге t2:


Репро: `python3 scripts/live_wizard_agentic_e2e.py` (мост 5.26 живой, bundle разложен).
Диагностика сохранена: сессия `wz_20260722_e74aa8`, build `build_20260722_1945_b660`

## Причина (подтверждена артефактами)

t1 (Excel) собирается и публикуется. t2 (PDF): Qwen генерит извлечение текста
**`basic_regex` по сырым байтам PDF без PDF-библиотеки** (`include(...)` в коде нет вовсе —
см. step_result: `"extraction_methods": ["basic_regex"]`, `files_used: [certificate.pdf]`,
0 записей). PDF от reportlab сжат (FlateDecode) → regex по байтам всегда 0. Все 3 попытки
ремонта повторяют тот же приём — «урок» не эскалирует к смене инструмента.

## Ожидаемое направление фикса (твой контур — builder prompt/repair)

1. В `_build_prompt`/`_code_artifact_prompt` (ui/wz_agentic.py): для входов `.pdf` —
   ЖЁСТКОЕ требование извлекать текст библиотекой через канон зависимостей
   `include("import pdfminer.six | pypdf | PyPDF2", ["extella-pip install …"])`;
   regex по сырым байтам PDF — запрещённый приём (объяснить почему: сжатые стримы).
2. Repair-урок: если приёмка отклонила с «0 записей из .pdf» — следующий ремонт обязан
   СМЕНИТЬ метод извлечения (библиотека), а не полировать тот же regex; иначе budget
   сгорает на идентичных попытках.
3. Регресс в preflight: сгенерённый код шага с `.pdf`-входом без PDF-библиотеки в
   `include` — не проходит (статическая проверка кода draft'а, аналог твоих lint-гейтов).

Верификация: `live_wizard_agentic_e2e.py` зелёный **дважды подряд** (кодоген стохастичен).

## Замечания в твой бэклог (не блокеры)
