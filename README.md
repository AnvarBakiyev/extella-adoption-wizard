# Extella — Визард внедрения (Adoption Wizard)

Плагин-репозиторий для Extella: разворачивает **фабрику процессов** — визард ведёт клиента
от бизнес-боли к работающему ИИ-процессу (интервью → схема → стройка → запуск 24/7).

Ставится по ссылке на этот репозиторий: Extella-агент читает `INSTALL.md` и разворачивает всё
в аккаунт пользователя (эксперты, концепты, правила, агенты, интерфейс).

## Что внутри
- `experts/` — 38 экспертов `wz_*` (сессия, blueprint, стройка, деплой, планировщик,
  публикация в витрину, композитор, лог запусков, 5 коннекторов вывода:
  Telegram/Email/Slack/WhatsApp/SMS, 6 источников входа:
  Google Sheets/Bitrix24/PostgreSQL/MySQL/amoCRM/1С, онбординг). Каждый — с шапкой
  `# expert: / # description: / # params:`.
- `concepts/` — концепты методологии и каталог возможностей.
- `catalog/` — каталог способностей + живые базы знаний (`catalog.json`, `knowledge_packs`).
- `rules/` — канон платформы для агентов визарда (что можно/нельзя).
- `agents/` — инструкции двух агентов: **Визард** (проектирует) и **Строитель** (строит).
- `ui/` — интерфейс визарда (`wizard.html`) + локальный мост (`server.py`).
- `docs/` — план рефакторинга и внутренние документы; `INSTALL.md` — инструкция для агента Extella,
  `HANDOUT.md` — памятка для коллег, `HANDOFF_TOOLBAR_v1.2.md` — контракт карточки для тулбара.
- `extella-plugin.json` — манифест плагина · `install.py`/`diag.py` — установщик и диагностика.

## Безопасность
В репозитории **нет секретов**. Токены/пароли эксперты читают из зашифрованного сейфа (vault)
на устройстве в рантайме. Репозиторий можно публиковать.

## Обновление
Пользователь заново прогоняет установку из обновлённого репозитория — эксперты/концепты/правила
перезаписываются свежими; мост дополнительно умеет само-обновляться по подписанному каналу релизов.
