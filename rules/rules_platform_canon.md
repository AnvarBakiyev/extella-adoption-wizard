# Rules — канон платформы для агентов визарда

Правила, которым обязаны следовать агенты Визард и Строитель. Нарушение = сломанный прод.

---

## rule_id: wz_model
Все клиентские агенты — платформенная модель **Qwen 3.7**: API-создание разрешено только с
явными `provider=alibaba` и моделью `qwen3.7-*`; UI-копия базового Qwen — допустимый фолбэк.
BYOK / gpt-4o для клиентских агентов запрещён (нечестный тест).

---

## rule_id: wz_scoping
Эксперты и концепты изолированы per-agent; всё общее — сохранять `global: true` и вызывать с
`global: true`. Один expert_name не должен жить в нескольких скоупах (дубль = недетерминированный
запуск).

---

## rule_id: wz_rest
REST-база `https://api.extella.ai`, заголовки X-Auth-Token / X-Profile-Id: default / X-Agent-Id.
Пути в единственном числе (`/api/agent/*`, `/api/expert/*`); список экспертов — `/api/experts_db/list`.

---

## rule_id: wz_experts_hosting
nohup-эксперты: сырой python БЕЗ `$extens`/def/return сверху; `{{placeholder}}` подставляются
только для явно переданных параметров → в коде фолбэки `if not X or X.startswith("{{")`.
Секреты — фолбэк из `~/extella_wizard/app/config.json`, иначе честный fail. Зависимости — только
через `include("import X", ["extella-pip install X"])`.

---

## rule_id: wz_agents_api
Правило «один инструмент за ход» (баг склейки аргументов); длинные задачи дробить (лимит
50 итераций/ход); HTTP 500/timeout ≠ провал — проверять артефактами; сообщения агентам делать
самодостаточными (previous_response_id теряется).

---

## rule_id: wz_change_control
Изменения процессов — только через Строителя с записью решения в сессию. Продовые агенты
заморожены. Письма/внешняя запись — только черновики, отправляет человек.

---

## rule_id: wz_anti_hallucination
Никогда не выдумывать факты, причины, диагнозы, шаги, URL. Показывать реальные ответы инструментов
дословно. При незнании — честно «не знаю, уточню» и предложить проверяемое действие.
